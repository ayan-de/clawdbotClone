## 🔐 Security Considerations

### Orbit Token Flow

**Why this is secure:**

1. **Token-based authentication** - Desktop TUI uses token instead of username/password
2. **Token generated by web app** - After user authorization
3. **Token is ephemeral** - Can be revoked/rotated
4. **No credentials in code** - Token passed securely via CLI

## 📋 How Desktop TUI Uses Token

### Token Storage

**From packages/desktop/src/config.ts:**

```typescript
interface DesktopConfig {
    bridgeUrl: string;       // Bridge server URL
    token: string;          // Authentication token from website
    desktopName?: string;    // Display name for sessions
    workspace?: string;      // Working directory for commands
    debug?: boolean;         // Debug mode
}

function loadConfig(cliArgs: any): DesktopConfig {
    // Load from CLI args: --token <value>
    // Or from config file: ~/.orbit/config.json
    const token = cliArgs.token || config.token;

    return {
        bridgeUrl,
        token,  // ← This token is used for authentication
        desktopName,
        workspace,
        debug,
    };
}
```

### Token Transmission

**From packages/desktop/src/client.ts:**

```typescript
// When connecting to Bridge Server
await this.socket.emit('desktop:connect', {
    token: this.config.token,  // ← Send token for authentication
    name: this.config.desktopName,
    workspace: this.config.workspace,
});

// Bridge Server validates token

## 🔄 How Token Generation Works

### In Web Dashboard (apps/web)

**User flow:**

1. User signs up/logs in to Web Dashboard
2. User goes to Settings → Desktop Authorization
3. User clicks "Connect to Telegram" (or other platform)
4. User enters username
5. Web Dashboard creates authorization request
6. User is redirected to Telegram app
7. User approves authorization
8. Telegram redirects back with auth code
9. Web Dashboard exchanges auth code for token
10. Web Dashboard displays token: "orbit-sfsdfs" (or similar)
11. User copies token

**Token format:**
- **Type**: Random string (UUID or similar)
- **Expiry**: Can be set (e.g., 24 hours, never)
- **Scope**: Linked to specific user and platform
- **Revocable**: User can revoke from settings
